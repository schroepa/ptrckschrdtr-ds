# Astro Docs-Site Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build an Astro documentation site at ds.ptrckschrdtr.de that visually documents all design system components using the DS's own tokens and components.

**Architecture:** Astro is installed in the repo root. `src/` holds pages and layouts. `public/` is Astro's static-assets folder — `public/r/` (registry output) is automatically included in the build output (`dist/`). Vercel runs `registry:build && astro build` and serves `dist/`.

**Tech Stack:** Astro 5, plain CSS (DS tokens only), no additional frameworks

**Files:**


- [ ] **Step 1.2: Update package.json scripts**

Replace the entire `"scripts"` block in `package.json` with:

```json
{
  "name": "ptrckschrdtr-ds",
  "version": "0.1.0",
  "description": "Personal design system registry by @ptrckschrdtr. Multi-brand, framework-agnostic, built on shadcn registry.",
  "homepage": "https://ds.ptrckschrdtr.de",
  "scripts": {
    "dev": "astro dev",
    "build": "astro build",
    "preview": "astro preview",
    "registry:build": "shadcn build",
    "registry:build:watch": "shadcn build --watch",
    "build:all": "npm run registry:build && npm run build"
  },
  "dependencies": {
    "astro": "latest"
  },
  "devDependencies": {
    "shadcn": "latest"
  }
}
```

- [ ] **Step 1.3: Create astro.config.mjs**

Create `astro.config.mjs` in the project root:
